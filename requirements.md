A Complex Engineering Problem (CEP) involves multiple interacting components,
conflicting constraints, and requires in-depth technical knowledge, analysis, and decisionmaking. In Full Stack Development projects, students must go beyond simple CRUD
applications and demonstrate real-world engineering complexity.
WP1: Depth of Knowledge Required
Projects must involve core engineering concepts such as API design, database modeling,
authentication mechanisms, and system architecture. Students should apply firstprinciples thinking rather than copying templates.
WP2: Conflicting Requirements
Projects should include trade-offs such as performance vs security, scalability vs cost,
and usability vs complexity.
WP3: Depth of Analysis
The problem should not have a straightforward solution. Students must make design
decisions such as choosing appropriate technologies and architectures.
WP4: Familiarity of Issues
Projects should involve novel or less familiar challenges such as real-time systems, API
integrations, or advanced frontend state management.
WP5: Applicable Codes
Students must follow industry standards like REST API design, secure authentication
(JWT), validation, and coding best practices.
WP6: Stakeholder Involvement
Projects must consider multiple stakeholders such as users, admins, and external systems,
each with different requirements.
WP7: Interdependence
The system should consist of interconnected components such as frontend, backend,
database, APIs, and deployment infrastructure.
Conclusion:
A well-designed mini project should clearly demonstrate all CEP characteristics. Such
projects are suitable for showcasing in resumes and reflect strong engineering capability.
This problem requires application of software engineering principles such as system
design, modular architecture, and scalable backend development.
